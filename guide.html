Guide d'utilisation et consignes de test

Ce playbook est en version beta. Il est fonctionnel mais en cours d'amelioration. Vos retours sont essentiels pour le perfectionner. Chaque remarque, bug ou idee d'amelioration compte.

A quoi sert le Playbook ?

Le Playbook est votre outil central pour structurer, suivre et piloter tous les process operationnels de Face Soul Yoga. Il remplace les listes de taches dispersees, les fichiers Excel et les post-its mentaux.

Ce que le Playbook vous permet :

  • Creer un process a partir d'un modele pre-rempli (11 modeles disponibles)
  • Suivre l'avancement avec des barres de progression et des indicateurs visuels
  • Savoir qui fait quoi grace au systeme RACI sur chaque etape
  • Anticiper les retards — les etapes en retard passent en rouge automatiquement
  • Voir la vision globale avec la Timeline (Gantt)
  • Reutiliser — un process termine peut etre duplique pour la prochaine fois

Les 4 vues

1
Playbook — La vue principale. Vous y creez vos process, consultez la liste, et ouvrez le detail de chaque process pour cocher les etapes.
2
Mes Taches — Votre vue personnelle. Filtrez par votre prenom pour voir toutes vos taches de tous les process, triees par date. Chaque tache indique si vous devez la faire ou la valider. Cliquez sur une tache pour voir le detail et ajouter des notes.
3
Timeline — La vision globale de tous vos process dans le temps. Vous pouvez filtrer pour voir un seul process en detail ou tous en vue d'ensemble.
4
Modeles — Consultez les 11 modeles disponibles avec toutes leurs etapes, le RACI et les timings. C'est votre reference pour comprendre comment chaque process est structure.

Les indicateurs visuels

  • Vert — Etape terminee
  • Rouge — Etape en retard (date depassee, pas encore faite)
  • Gris — Etape a venir

Comment utiliser le Playbook au quotidien

Routine quotidienne (2 minutes)

  1. Ouvrir le Playbook
  2. Aller dans Mes Taches
  3. Filtrer par votre prenom
  4. Regarder les taches du jour et celles en retard (rouge)
  5. Cocher ce qui est fait

Routine hebdomadaire (10 minutes)

  1. Ouvrir la Timeline pour voir la vue d'ensemble
  2. Verifier qu'aucun process n'a trop de retard
  3. Regarder la semaine a venir dans Mes Taches
  4. Ajuster les timings si necessaire (dans le detail du process)

Ce que j'attends de vous (beta test)

Objectif : tester le playbook pendant 2 semaines en conditions reelles, puis faire un retour structure pour ameliorer l'outil.

Pendant la phase beta, merci de :

  1. Creer au moins 2 process a partir des modeles (par exemple : la migration Circle en cours + un onboarding)
  2. Utiliser "Mes Taches" quotidiennement — cocher vos taches, regarder ce qui est en retard
  3. Tester la Timeline — filtrer par process, verifier que c'est lisible
  4. Consulter les Modeles — verifier que les etapes, les timings et le RACI correspondent a votre realite
  5. Ajouter des descriptions aux taches quand necessaire (cliquer sur une tache dans Mes Taches)
  6. Noter tout ce qui ne va pas : bugs, etapes manquantes, timings incorrects, textes confus, fonctionnalites manquantes

Vos retours m'interessent sur :

  • Les modeles — Des etapes a ajouter, supprimer, renommer ? Des timings a ajuster ? Le RACI est correct ?
  • L'ergonomie — C'est facile a utiliser sur mobile ? Les informations sont claires ?
  • Les couleurs — Les personnes sont bien distinctes visuellement ?
  • La Timeline — Elle est lisible ? Utile ? Qu'est-ce qui manque ?
  • Les fonctionnalites manquantes — Qu'est-ce que vous aimeriez pouvoir faire que vous ne pouvez pas ?
  • Les bugs — Quelque chose qui ne marche pas comme prevu

Comment me faire vos retours

Envoyez-moi un message (email, WhatsApp, en seance) avec :

  • Ce que vous avez fait (ex: "j'ai cree un process migration")
  • Ce qui s'est passe (ex: "les dates ne correspondent pas")
  • Ce que vous attendiez (ex: "je pensais que J-90 serait le 15 janvier")

Pas besoin d'etre technique — decrivez simplement ce que vous voyez et ce qui vous gene.
